10.0 Introduction

Filters are commands that read in text as input, alter it, and then produce text as output. Standard input, standard output, standard error redirection, and pipes are used to apply filter commands and to work with text streams.

When a command executes without any errors, the output that is produced is known as standard out, also called stdout or STDOUT. By default, this output will be sent to the terminal where the command is being executed. The standard out is highlighted below.

sysadmin@localhost:~$ ls
Desktop Documents Downloads Music Pictures Public Templates Videos

When a command encounters an error, it will produce output that is known as standard error, also called stderr or STDERR. Like standard out, the standard error output is normally sent to the same terminal where the command is currently being executed. The standard error is highlighted below.

sysadmin@localhost:~$ fakecommand
-bash: fakecommand: command not found

Standard input, also called stdin or STDIN, normally comes from the keyboard with input provided by the user who runs the command. In this example, the sort command reads lines of input until Ctrl+D is pressed. The Ctrl+D key combination signals the end of input.

sysadmin@localhost:~$ sort
Banana
Peach
Apple
CTRL+D
Apple
Banana
Peach

Lastly, command pipelines are often used to make effective use of filter commands. In a command pipeline, the output of one command is sent to another command as input.

sysadmin@localhost:~$ history | less

10.1 Step 1

Standard output is redirected using the greater-than > character followed by the name of the file to which it will be written. Redirecting output using a single greater-than > character will create a new file, or overwrite the contents of an existing file with the same name. To redirect the output of the ls command to the ls_output.txt file, execute the following command:

ls > ls_output.txt
sysadmin@localhost:~$ ls > ls_output.txt

To view the contents of the newly created ls_output.txt file, use the following command:

cat ls_output.txt
sysadmin@localhost:~$ cat ls_output.txt
Desktop
Documents
Downloads
Music
Pictures
Public
Templates
Videos
ls_output.txt

10.2 Step 2

Redirecting standard output with two greater-than >> characters will result in the output of the command being appended to the end of a file if it does already exist.

To append the output of the date command to the ls_output.txt file created in the previous step, execute the following command:

date >> ls_output.txt
sysadmin@localhost:~$ date >> ls_output.txt
The number 1 is associated with the standard output file descriptor when used with the > and >> operators. The number is implied in the examples shown above; the example can also be written as date 1 >> ls_output.txt.

To view the contents of the ls_output.txt file, use the following command:

cat ls_output.txt

sysadmin@localhost:~$ cat ls_output.txt
Desktop
Documents
Downloads
Music
Pictures
Public
Templates
Videos
ls_output.txt
Thu Apr 30 13:21:38 UTC 2020

10.3 Step 3

Similar to standard output, standard error can also be redirected to files. The number 2 is associated with the standard error file descriptor. To redirect standard error while executing the find command, execute the following commands:

find /etc . -name hostname
find /etc . –name hostname 2> error.txt

sysadmin@localhost:~$ find /etc . -name hostname
/etc/hostname
find: '/etc/ssl/private': Permission denied
sysadmin@localhost:~$ find /etc . -name hostname 2> error.txt
/etc/hostname

The output of the find command shown above shows the difference when the error stream is redirected to a file. In the first instance of the find command, the Permission denied error is shown on the terminal, while in the second instance of the find command this error is redirected to the error.txt file.

To view the standard error from the previously executed command in the contents of the error.txt file, use the following command:

cat error.txt
sysadmin@localhost:~$ cat error.txt
find: '/etc/ssl/private': Permission denied

10.4 Step 4

To redirect both standard output and standard error of the find command and verify the output has been redirected, execute the following commands:

find /etc –name hostname > find_output.txt 2> find_error.txt
cat find_output.txt
cat find_error.txt

sysadmin@localhost:~$ find /etc -name hostname > find_output.txt 2> find_error.txt
sysadmin@localhost:~$ cat find_output.txt
/etc/hostname
sysadmin@localhost:~$ cat find_error.txt
find: '/etc/ssl/private': Permission denied

10.5 Step 5

To discard the output of any stream instead of redirecting it to any file, redirect it to the /dev/null file:

find /etc -name hostname 2> /dev/null
sysadmin@localhost:~$ find /etc -name hostname 2> /dev/null
/etc/hostname
sysadmin@localhost:~$

10.6 Step 6

To redirect both standard output and standard error of the find command to the same file, execute the following command:

find /etc –name hostname > find_output.txt 2>&1
cat find_output.txt

sysadmin@localhost:~$ find /etc -name hostname > find_output.txt 2>&1
sysadmin@localhost:~$ cat find_output.txt
/etc/hostname
find: '/etc/ssl/private': Permission denied

In the command above, the 2>&1 expression is used to send stderr to the same location as stdout.

10.7 Step 7

An alternative command to redirect both standard output and standard error:

find /etc –name hostname &> find_output.txt
cat find_output.txt

sysadmin@localhost:~$ find /etc -name hostname &> find_output.txt
sysadmin@localhost:~$ cat find_output.txt
/etc/hostname
find: '/etc/ssl/private': Permission denied

In the examples above, the notation &> will redirect all output.

10.8 Step 8

The < symbol is used for standard input redirection and used to pass the contents of a file as input to a command. Execute the following commands to view the contents of the /etc/hosts file, then translate all the lower-case characters in the /etc/hosts file to upper-case characters:

cat /etc/hosts
tr 'a-z' 'A-Z' < /etc/hosts

10.9 Step 9

Both input as well as output redirection can be used together in the same command. To translate all the lower-case characters in the /etc/hosts file to upper-case characters and save the output in the ~/myhosts file, execute the following command:

tr 'a-z' 'A-Z' < /etc/hosts > ~/myhosts
sysadmin@localhost:~$ tr 'a-z' 'A-Z' < /etc/hosts > ~/myhosts

To view the redirected standard input and standard output in the contents of the ~/myhosts file, use the following command:

cat ~/myhosts
sysadmin@localhost:~$ cat ~/myhosts

10.11 Step 11

Command pipelines are often used to make effective use of filter commands. In a command pipeline, the output of one command is sent to another command as input. In Linux and most operating systems, the vertical bar or pipe | character is used between two commands to represent a command pipeline.

Redirect the text output by the history command to the grep command as input by using the following command:

history | grep "cat"
sysadmin@localhost:~$ history | grep "cat"
    4  cat ls_output.txt
    6  cat ls_output.txt
   10  cat error.txt
   12  cat find output.txt
   13  cat find_output.txt
   14  cat find_error.txt
   19  cat find_output.txt
   20  cat /etc/hosts
   23  cat ~/myhosts
   25  history | grep "cat"

In the example above, the grep command matches the cat strings in the command history and sends its output to standard out.

10.12 Step 12

The pipe | symbol can be used multiple times within a command. To extract some fields from the /etc/passwd file, sort them, and finally eliminate duplicate lines, execute the following command:

cut –f7 –d: /etc/passwd | sort | uniq -c
sysadmin@localhost:~$  cut -f7 -d: /etc/passwd | sort | uniq -c
      3 /bin/bash
      1 /bin/sync
     24 /usr/sbin/nologin

The previous command extracted the seventh field from the /etc/passwd file with the cut command, then sorted those lines with the sort command, and finally, eliminated duplicate lines with the uniq command.

The -c option to the uniq command will prefix lines in the output with the total number of occurrences of that line in the file specified.

10.13 Step 13

The pipe | symbol can also be combined with redirection. To count the number of occurrences of man in the command history, and redirect the result to the count.txt file, execute the following command:

history | grep ls | wc –l > count.txt
cat count.txt

sysadmin@localhost:~$ history | grep ls | wc -l > count.txt
sysadmin@localhost:~$ cat count.txt
7
The number of occurrences of ls in the command history may differ in our virtual environment.
The wc command used with the -l option will print a count of lines in a file. In the command above, the grep command is piped to the wc command, so the wc command only counts lines with the ls string.

10.16 Step 16

The xargs command is used to read data from standard input and execute a command based on that input. In this step, the xargs command uses the touch command to create four new files in the home directory.

echo 'x1 x2 x3 x4' | xargs touch
ls

sysadmin@localhost:~$ echo 'x1 x2 x3 x4' | xargs touch
sysadmin@localhost:~$ ls
Desktop    Music     Templates  error.txt        homedir.txt    x1  x4
Documents  Pictures  Videos     find_error.txt   ls_output.txt  x2
Downloads  Public    count.txt  find_output.txt  myhosts        x3

10.17 Step 17

Remove the four files created in the previous step by using the rm command with the xargs command:

echo 'x1 x2 x3 x4' | xargs rm
ls

sysadmin@localhost:~$ echo 'x1 x2 x3 x4' | xargs rm
sysadmin@localhost:~$ ls
Desktop    Music     Templates  error.txt        homedir.txt
Documents  Pictures  Videos     find_error.txt   ls_output.txt
Downloads  Public    count.txt  find_output.txt  myhosts

10.19 Step 19

A delimiter can be set using the -d option with the xargs command. To view the contents of the /etc directory beginning with ca with all instances of the dash - character replaced with a space character, use the following commands:

ls /etc | grep ^ca
ls /etc | grep ^ca | xargs -d '-'

sysadmin@localhost:~$ ls /etc | grep ^ca
ca-certificates
ca-certificates.conf
calendar
sysadmin@localhost:~$ ls /etc | grep ^ca | xargs -d '-'
ca certificates
ca certificates.conf
