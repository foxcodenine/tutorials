14.3 Step 3

To disable an Ethernet interface, it can be brought down with the ifconfig [device] down command. Disable the ens3 interface and check the status of the interface by executing the following commands:

ifconfig ens3 down
ifconfig ens3
[root@centos ~]# ifconfig ens3 down
[root@centos ~]# ifconfig ens3
ens3: flags=4098<BROADCAST,MULTICAST> mtu 1500
      ether 2c:fd:a1:ac:45:32  txqueuelen 1000  (Ethernet)
      RX packets 10  bytes 1998 (1.9 KiB)
      RX errors 0  dropped 0  overruns 0  frame 0
      TX packets 94  bytes 6880 (6.7 KiB)
      TX errors 0  dropped 0 overruns 0  carrier 0  collitions 0

The second ifconfig command output shows that there are no UP and RUNNING flags, indicating that the interface is down.

14.4 Step 4

To enable an Ethernet interface, it can be brought up with the ifconfig [device] up command. Enable the ens3 interface and check the status of the interface by executing the following commands:

ifconfig ens3 up
ifconfig ens3
[root@centos ~]# ifconfig ens3 up
[root@centos ~]# ifconfig ens3
ens3: flags=4163<UP,BROADCAST,RUNNING,MULTICAST> mtu 1500
      inet 10.0.2.3 netmask 255.255.255.0 broadcast 10.0.2.255
      inet6 fe80::2efd:a1ff:feac:4532  prefixlen 64  scopeid 0x20<link> 
      inet6 2001:9b0:104:42::103 prefixlen 128  scopeid 0x0<global>
      ether 2c:fd:a1:ac:45:32  txqueuelen 1000  (Ethernet)
      RX packets 12  bytes 2400 (2.3 KiB)
      RX errors 0  dropped 0  overruns 0  frame 0
      TX packets 110  bytes 8374 (8.1 KiB)
      TX errors 0  dropped 0 overruns 0  carrier 0  collitions 0

The state of the ens3 interface is now UP.

14.5 Step 5

To view the networking configuration, tools such as the ifconfig command and the iproute2 suite of commands can be used. This portion of the lab will use the iproute2 suite of tools instead of the ifconfig command, which is being phased out in newer Linux distributions.

The iproute2 tools will be covered in complete detail later in the course.

14.6 Step 6

To disable an Ethernet interface, it can be brought down with the ip link set command. Disable the interface and check the status of the interface by executing the following commands:

ip address show ens3
ip link set ens3 down
ip address show ens3
[root@centos ~]# ip address show ens3
2: ens3: <UP,BROADCAST,RUNNING,MULTICAST> mtu 1500 qdisc pfifo_fast state UP group default qlen 1000
    link/ether 2c:fd:a1:ac:45:32 brd ff:ff:ff:ff:ff:ff
      inet 10.0.2.3/24 brd 10.0.2.255 scope global noprefixroute dynamic ens3
        valid_lft 86389sec preferred_lft 86389sec
      inet6 2001:9b0:104:42::103/128 scope global noprefixroute dynamic ens3
        valid_lft 3570sec preferred_lft 3570sec
      inet6 fe80::2efd:a1ff:feac:4532/64 scope link noprefixroute
        valid_lft forever preferred_lft forever
[root@centos ~]# ip link set ens3 down
[root@centos ~]# ip address show ens3
2: ens3: <BROADCAST,MULTICAST> mtu 1500 qdisc pfifo_fast state DOWN group default qlen 1000
    link/ether 2c:fd:a1:ac:45:32 brd ff:ff:ff:ff:ff:ff

Note in the output of the third ip address command that there are no IP addresses displayed and the state of the ens3 interface is DOWN.

14.7 Step 7

To enable the Ethernet interface, if it is not configured to start at boot time, the interface can be brought up with the ip link set command. Execute the following commands:

ip link set ens3 up 
ip address show ens3
[root@centos ~]# ip link set ens3 up
[root@centos ~]# ip address show ens3
2: ens3: <BROADCAST,MULTICAST,UP,LOWER,UP> mtu 1500 qdisc pfifo_fast state UP group default qlen 1000
      link/ether 2c:fd:a1:ac:45:32 brd ff:ff:ff:ff:ff:ff
      inet 10.0.2.3/24 brd 10.0.2.255 scope global noprefixroute dynamic ens3
        valid_lft 86389sec preferred_lft 86389sec
      inet6 2001:9b0:104:42::103/128 scope global noprefixroute dynamic ens3
        valid_lft 3570sec preferred_lft 3570sec
      inet6 fe80::2efd:a1ff:feac:4532/64 scope link noprefixroute
        valid_lft forever preferred_lft forever

The state of the ens3 interface is now UP.

14.9 Step 9

Create a copy of the network interface configuration file in your home directory by executing the following command:

cp /etc/sysconfig/network-scripts/ifcfg-ens3 /root
[root@centos ~]# cp /etc/sysconfig/network-scripts/ifcfg-ens3 /root
This step is designed to provide a backup of the original file in the event that you make a mistake while editing the file.

14.10 Step 10

Execute the following command to modify the network interface configuration file:

vi /etc/sysconfig/network-scripts/ifcfg-ens3
NAME="ens3"
ONBOOT=yes
NETBOOT=yes
IPV6INIT=yes
BOOTPROTO=dhcp
TYPE=Ethernet
 
 
 
 
"/etc/sysconfig/network-scripts/ifcfg-ens3" 6L, 77C

14.11 Step 11

Modify the content of the file to create a static configuration. Use the vi insert i command to begin editing the file with the following configuration:

DEVICE=ens3 
ONBOOT=yes 
BOOTPROTO=static 
TYPE=Ethernet 
IPADDR=192.168.1.11 
NETMASK=255.255.255.0 
GATEWAY=192.168.1.1 
Esc
:wq

The entire contents of the file should look like the file shown below:

DEVICE=ens3
ONBOOT=yes
BOOTPROTO=static
TYPE=Ethernet
IPADDR=192.168.1.11
NETMASK=255.255.255.0
GATEWAY=192.168.1.1
 
 
 
 
 
"/etc/sysconfig/network-scripts/ifcfg-eth0" 9L, 146C written
[root@centos ~]#

14.12 Step 12

Although this new configuration could be made active by bringing the interface down and back up again (or by restarting the system), you can also use the init script to restart the network service. Execute the following command:

/etc/init.d/network restart
[root@centos ~]# cp /etc/init.d/network restart
Restarting network (via systemctl):                       [  OK  ]
[root@centos ~]#

The output indicates that the network is being restarted via the systemctl command; should the restart fail, it will return an error message to assist with troubleshooting.

14.13 Step 13

Execute the following command to verify that the configuration of the device matches the static configuration used:

ip address show ens3
[root@centos ~]# ip address show ens3
2: ens3: <BROADCAST,MULTICAST,UP,LOWER,UP> mtu 1500 qdisc pfifo_fast state UP group default qlen 1000
	link/ether 2c:fd:a1:ac:45:32 brd ff:ff:ff:ff:ff:ff
	inet 192.168.1.11/24 brd 192.168.1.255 scope global nprefixroute ens3
       valid_lft preferred_lft forever
	inet6 fe80::2efd:a1ff:feac:4532/64 scope link 
       valid_lft forever preferred_lft forever

14.14 Step 14

Execute the following command to verify that the routing table shows the Gateway value entered in the static configuration as the default router:

ip route show
[root@centos ~]# ip route show
default via 192.168.1.1 dev ens3 proto static metric 100
192.168.1.0/24 dev ens3 proto kernel scope link src 192.168.1.11 metric 100

14.16 Step 16

Restart the network service by using the service command instead of executing the init script directly:

service network restart
[root@centos ~]# service network restart
Restarting network (via systemctl):                       [  OK  ]
[root@centos ~]#

Recall that the service command acts as a front end to the init scripts. It is not available on all Linux distributions.

14.19 Step 19

Typically, after changing the name of a system it is rebooted, so any services or running processes will be able to use the new name. Execute the following command to reboot the system:

reboot

After shutting down everything, and starting back up again, the hostname is changed as is seen in the following login prompt:

[root@centos ~]# reboot

14.20 Step 20

On most systems, the IP address for the localhost hostname is defined in the /etc/hosts file. Display the contents of this file by executing the following command:

cat /etc/hosts
[root@cserver ~]# cat /etc/hosts
127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
::1         localhost localhost.localdomain localhost6 localhost6.localdomain6

14.21 Step 21

Not only can hostnames like localhost or centos be used, but other hostnames like cserver or cserver.example.com can be used for connectivity if they are added to the /etc/hosts file. Execute the following command to append an entry to the /etc/hosts file:

echo '192.168.1.3 cserver cserver.example.com' >> /etc/hosts
[root@cserver ~]# echo '192.168.1.3 cserver cserver.example.com' >> /etc/hosts

14.22 Step 22

Execute the following command to attempt to connect using this new hostname:

ping -c5 cserver
[root@centos ~]# ping -c5 cserver
PING cserver (192.168.1.3) 56(84) bytes of data.
PING cserver.example.com (10.0.2.3) icmp_seq=1 Destination Host Unreachable
PING cserver.example.com (10.0.2.3) icmp_seq=2 Destination Host Unreachable
PING cserver.example.com (10.0.2.3) icmp_seq=3 Destination Host Unreachable
PING cserver.example.com (10.0.2.3) icmp_seq=4 Destination Host Unreachable
PING cserver.example.com (10.0.2.3) icmp_seq=5 Destination Host Unreachable
 
--- cserver ping statistics ---
5 packets transmitted, 0 received, 100% packet loss, time 3999ms

Currently, you only have a configured network interface on the 127.0.0.0/255.0.0.0 network. Because the address 192.168.1.3 is not on this network, the system will be unable to reach the IP address associated with cserver without a router.

The ping command is used to check the connectivity to a host. This command will be covered in detail in a later chapter.

14.25 Step 25

Configuration files for network interfaces in Debian-derived systems, such as Ubuntu, are located in the /etc/netplan directory. The file in our virtual machine is located at /etc/netplan/01-netcfg.yaml. Execute the following command to view this file:

cat /etc/netplan/01-netcfg.yaml 
[root@ubuntu ~]# cat /etc/netplan/01-netcfg.yaml
# This file describes the network interfaces available on your system
# For more information, see netplan(5).
network:
 version: 2
 rendere: networkd
 ethernets:
  ens3: 
   dhcp4: yes

Additional *.yaml files can be placed in this directory to tell the systemd-networkd system service how to configure network interfaces.
