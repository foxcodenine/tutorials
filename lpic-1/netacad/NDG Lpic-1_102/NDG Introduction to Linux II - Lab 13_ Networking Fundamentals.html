13.1 Step 1

For this lab you will be using the Ubuntu virtual machine.

Log in as the sysadmin user. When prompted for ubuntu login:, enter sysadmin. When prompted for a password, enter netlab123:

Ubuntu 18.04.3 LTS ubuntu tty1
 
ubuntu login: sysadmin
Password:
Welcome to Ubuntu 18.04.3 LTS (GNU/Linux 4.15.0-55-generic x86_64)

13.2 Step 2

To display IP address information on the command line, the ifconfig and ip address commands can be used. The ifconfig command is becoming obsolete (deprecated) in some Linux distributions and is being replaced with a form of the ip command, specifically ip address.

View the current configuration of all interfaces by executing the ip address show command:

ip address show
sysadmin@ubuntu:~$ ip address show
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
    inet 127.0.0.1/8 scope host lo
      valid_lft forever preferred_lft forever
2: ens3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_code1 state UP group default qlen 1000
    link/ether 2c:fd:a1:53:8b:82 brd ff:ff:ff:ff:ff:ff
    inet 10.0.2.2/24 brd 10.0.2.255 scope global dynamic ens3
      valid_lft 86220sec preferred_lft 86220sec
    inet6 2001:9b0:104:42::102/128 scope global dynamic noprefixroute
      valid_lft 3491sec preferred_lft 3491sec
    inet6 fe80:2efd:a1ff:fe53:8b82/64 scope link
      valid_lft forever preferred_lft forever

The first field in the output shows the names of the interfaces.The lo interface represents the internal loopback interface. The loopback lo interface is designed to allow you to perform network-based commands that are intended to connect to your own system. It is useful primarily for standalone hosts that have the need to use network-based commands without actually being connected to a network.

The ens3 interface represents the only external Ethernet interface. The first line of the output for the Ethernet interface shows its HWaddr or MAC address (in this case, 2c:fd:a1:53:8b:82).

For each active interface, the second line of output should begin with inet addr: and displays the IPv4 address associated with that interface, possibly a broadcast address and the subnet mask for the interface. The subnet masks are shown in dotted decimal form, such as 255.255.255.0, or using what is called Variable Length Subnet Mask (VLSM) notation.

The line that begins with inet6 addr represents the IPv6 address for the interface, the number of bits of its subnet mask and its scope. The subnet masks are shown in the number of bits that are used to represent the network portion of the address, such as 64 or 128, or using what is called Classless Interdomain Routing (CIDR) notation. Not all interfaces are assigned an IPv6 address.

The rest of the information provided by this command is beyond the scope of this course.

The ip address command and other iproute2 utilities used to display IP address and route information in this lab will be covered in greater detail later in the course.

13.3 Step 3

To view where packets will be routed, you can display the routing table with the route command, on legacy systems, or the ip route show command. Since the route command has been superseded by the ip route command, the following steps will use the ip route command.

Execute the following command to view the routing table:

ip route show
sysadmin@ubuntu:~$ ip route show
default via 10.0.2.22 dev ens3 proto dhcp src 10.0.2.2 metric 100
10.0.2.0/24 dev ens3 proto kernel scope link src 10.0.2.2 
10.0.2.22 dev ens3 proto dhcp scope link src 10.0.2.2 metric 100

The output shows the local interface, ens3, as having IP address 10.0.2.0 and the default route is 10.0.2.2.

13.4 Step 4

New routes can be added on-the-fly with the ip route add command, but they are not persistent. To add a router device that has the IP address of 10.0.0.0 as a new default route, execute the following command. When prompted for a password, enter netlab123:

sudo ip route add 10.0.0.0/24 dev ens3
ip route show
sysadmin@ubuntu:~$ sudo ip route add 10.0.0.0/24 dev ens3
Password:
sysadmin@ubuntu:~$ ip route show
default via 10.0.2.2 dev ens3 proto dhcp src 10.0.2.2 metric 100
10.0.0.0/24 dev ens3 scope link
10.0.2.0/24 dev ens3 proto kernel scope link src 10.0.2.2 
10.0.2.22 dev ens3 proto dhcp scope link src 10.0.2.2 metric 100

This output means that any network connection that is not designed to be sent to a machine on the local network will now be sent to the default router. The new static default route will send any network traffic that isn’t destined for a machine on the local “private” 10.0.0.0/24 network out through a router configured on that network. This default router is commonly the router that connects your machine to the “outside world” (the Internet).

13.5 Step 5

For a list of common ports and services that may listen to network ports, view the contents of the /etc/services file. Execute the following command to display this file, press the Spacebar key to advance page by page through the file and press the Q key to quit when finished:

more /etc/services
sysadmin@ubuntu:~$ more /etc/services
# Network services, Internet style
#
# Note that it is presently the policy of IANA to assign a single well-known
# port number for both TCP and UDP; hence, officially ports have two entries
# even if the protocol doesn’t support UDP operations. 
#
# Updated from http://www.iana.org/assignments/port-numbers and other
# sources like http://www.freebsd.org/cgi/cvsweb.cgi/src/etc/services .
# New ports will be added on request if the have been officially assigned
# by IANA and used in the real-world or are needed by a debian package.
# If you need a huge list of used numbers please install the nmap package. 
 
tcpmux          1/tcp                          # TCP port service multiplexer
echo            7/tcp      
echo            7/udp        
discard         9/tcp           sink null
discard         9/udp           sink null
systat         11/tcp           users
daytime        13/tcp      
daytime        13/udp
netstat        15/tcp  
qotd           17/tcp           quote
msp            18/tcp                          # message send protocol
msp            18/udp 
--More--(4%)

The first column of output shows the name of the service.

tcpmux          1/tcp                         # TCP port service multiplexer

The second column shows the port/protocol,

tcpmux          1/tcp                         # TCP port service multiplexer

The third column may show an alias for the service name and the fourth column may show a comment.

tcpmux          1/tcp                         # TCP port service multiplexer

Type Ctrl+C to exit the more command:

Ctrl+c

13.6 Step 6

Internet hostnames and IP addresses are usually stored in the distributed Domain Name System (DNS) and can be queried by using the host, dig, and nslookup commands.

Use the host command with the host name ubuntu.com to query for its IP address that is stored in DNS by executing the following command:

host ubuntu.com
sysadmin@ubuntu:~$ host ubuntu.com

ubuntu.com has address 91.189.89.115
ubuntu.com has address 91.189.89.118
ubuntu.com has address 91.189.90.58
ubuntu.com has address 91.189.89.103
ubuntu.com has address 91.189.90.59
ubuntu.com mail is handled by 10 mx.ubuntu.com.

13.7 Step 7

DNS query commands like host depend on nameserver entries in the /etc/resolv.conf file. Execute the following command to view the contents of this file:

cat /etc/resolv.conf
sysadmin@ubuntu:~$ cat /etc/resolv.conf  
# This file is managed by man:systemd-resolved(8). Do not edit.
#           
# This is a dynamic resolv.conf file for connecting local clients to the 
# internal DNS stub resolver of systemd-resolved. This file lists all
# configured domains.
#
# Run “systemd-resolve --status” to see details about the uplink DNS servers
# currently in use. 
#
# Third party programs must not access this file directly, but only through the
# symlink at /etc/resolv.conf. To manage man:resolv.conf(5) in a different way,
# replace this symlink by a static file or a different symlink. 
#
# See man :systemd-resolved.service(8) for details about the supported modes of
# operation for /etc/resolv.conf.
                                                 
nameserver 127.0.0.53                                                           
options edns0

13.8 Step 8

Reverse DNS queries look up the hostname when provided the IP address of the host. Use the following command to perform a reverse DNS query, returning the host name ubuntu.com:

sysadmin@ubuntu:~$ host 91.189.89.110                                  
110.89.189.91.in-addr.arpa domain name pointer www-ubuntu-com.nuno.canonical.com.

Recall that the IP address used to perform the reverse DNS query was obtained in the output of the host ubuntu.com command in a previous step.

sysadmin@ubuntu:~$ host ubuntu.com
