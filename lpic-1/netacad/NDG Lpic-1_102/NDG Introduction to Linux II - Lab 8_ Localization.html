8.3 Step 3

Change the LC_NUMERIC locale variable to C.UTF-8 by executing the following command:

export LC_NUMERIC="C.UTF-8"
sysadmin@localhost:~$ export LC_NUMERIC="C.UTF-8"

Note that the tzselect command makes no changes to the system. A regular user can use this information to change the time zone settings for their account, as demonstrated in the following steps.

8.9 Step 9

To make this change to your current environment, first execute the following command to display the current time zone settings:

echo $TZ
date

sysadmin@localhost:~$ echo $TZ                                                  
                                                                                
sysadmin@localhost:~$ date                                                      
Fri Nov 15 03:36:55 UTC 2019

The first command verifies that no custom time zone is set. The second command verifies that the system default time zone is being used (UTC).

8.10 Step 10

Now, set the TZ variable by executing the following command:

export TZ="Asia/Tokyo"
sysadmin@localhost:~$ export TZ="Asia/Tokyo"

8.11 Step 11

Next, verify these changes by executing the following commands:

echo $TZ
date

sysadmin@localhost:~$ echo $TZ                                                  
Asia/Tokyo                                                                      
sysadmin@localhost:~$ date                                                      
Fri Nov 15 12:38:02 JST 2019

8.12 Step 12

To modify the system time zone, begin by executing the following command to switch to the root account. When prompted, provide the password netlab123

su –
sysadmin@localhost:~$ su -                                                 
Password:                                                                       
root@localhost:~#

8.13 Step 13

Next, execute the date command to verify the current time zone:

date
root@localhost:~# date                                                          
Fri Nov 15 03:40:14 UTC 2019

This output demonstrates that the current time zone is UTC.

Recall that the system time zone is defined in the /etc/localtime file. This is a binary file that should not be viewed directly.

8.14 Step 14

Before changing the /etc/localtime file, make a backup copy by executing the following command:

cp /etc/localtime /tmp
root@localhost:~# cp /etc/localtime /tmp

In the event that an error occurs when changing the time zone, the backup file would provide an easy method to recover the system time zone.

8.15 Step 15

Next, link the /etc/localtime file to a time zone file located in the /usr/share/zoneinfo directory structure. Execute the following command:

ln –sf /usr/share/zoneinfo/Asia/Tokyo /etc/localtime
root@localhost:~# ln –sf /usr/share/zoneinfo/Asia/Tokyo /etc/localtime

8.16 Step 16

Verify the time zone has changed by executing the following command:

date
root@localhost:~# date
Fri Nov 15 03:43:34 JST 2019
This lab is complete.
