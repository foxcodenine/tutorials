15.3 Step 3

To disable an Ethernet interface on a legacy system, it can be brought down with the ifdown command. Disable the ens3 interface and check the status of the interface by executing the following commands:

ifdown ens3
ifconfig ens3
[root@centos ~]# ifdown ens3
Device ‘ens3’ successfully disconnected.
[root@centos ~]# ifconfig ens3
ens3: flags=4163<UP,BROADCAST,RUNNING,MULTICAST> mtu 1500 
      ether 2c:fd:a1:ac:45:32  txqueuelen 1000  (Ethernet)
      RX packets 33  bytes 6751 (6.5 KiB)
      RX errors 0  dropped 0  overruns 0  frame 0
      TX packets 46  bytes 4564 (4.4 KiB)
      TX errors 0  dropped 0 overruns 0  carrier 0  collitions 0

The ifconfig output displayed no IPv4 addresses for the ens3 interface, indicating that the interface is down.

15.4 Step 4

To enable an Ethernet interface on a legacy system, it can be brought up with the ifdown command. Enable the ens3 interface and check the status of the interface by executing the following commands:

ifup ens3
ifconfig ens3
[root@centos ~]# ifup ens3
Connection successfully activated (D-Bus active path: /org/freedesktop/NetworkManager/ActiveConnection/3)
[root@centos ~]# ifconfig ens3
ens3: flags=4163<UP,BROADCAST,RUNNING,MULTICAST> mtu 1500 
      inet 10.0.2.3 netmask 255.255.255.0 broadcast 10.0.2.255
      inet6 fe80::2efd:a1ff:feac:4532  prefixlen 64  scopeid 0x20<link> 
      inet6 2001:9b0:104:42::103 prefixlen 128  scopeid 0x0<global>
      ether 2c:fd:a1:ac:45:32  txqueuelen 1000  (Ethernet)
      RX packets 12  bytes 2400 (2.3 KiB)
      RX errors 0  dropped 0  overruns 0  frame 0
      TX packets 46  bytes 4564 (4.4 KiB)
      TX errors 0  dropped 0 overruns 0  carrier 0  collitions 0

Notice the inet and inet6 IP addresses in the output of the ifconfig command after bringing the ens3 interface back up?

15.5 Step 5

To see if the module for the network device has been loaded, you can search the output of the lsmod command with the grep command. Execute the following command to see if the e1000 module has been loaded for the network device:

lsmod | grep 'e1000'
[root@centos ~]# lsmod | grep ‘e1000’
e1000                 137624  0

The e1000 module provides a driver for Intel networking cards that supports 1,000 Mbps (Megabits per second) of data transportation.

Most modern network cards support this data transfer rate, but in the real world you may come across network cards that only support 100 Mbps. These legacy Intel network cards are supported by the e100 module. Another more modern driver is the e1000e module, which supports PCI-Express adapters whereas the e1000 supports traditional PCI adapters.

15.12 Step 12

Be aware that there are circumstances where the ping command will not succeed. Execute the following ping command, which should fail:

ping -c3 10.0.2.1
root@ubuntu ~# ping -c3 10.0.2.1
PING 10.0.2.1 (10.0.2.1) 56(84) bytes of data.
From 10.0.2.1 icmp_seq=1 Destination Host Unreachable
From 10.0.2.1 icmp_seq=2 Destination Host Unreachable
From 10.0.2.1 icmp_seq=3 Destination Host Unreachable
 
--- 10.0.2.1 ping statistics ---
3 packets transmitted, 0 received, +3 errors, 100% packet loss, time 2000ms
pipe 3

In this case, the host may actually be unreachable because it is either not at that address, may not be returning ping requests or may have other networking issues.

15.13 Step 13

Another reason that connectivity may fail is due to the lack of a route to the network that the host is located on. Executing the following ping command should also fail:

ping -c3 192.168.1.1
root@ubuntu ~# ping -c3 192.168.1.1
PING 192.168.1.1 (192.168.1.1) 56(84) bytes of data.
From 10.0.2.2 icmp_seq=1 Destination Host Unreachable
From 10.0.2.2 icmp_seq=2 Destination Host Unreachable
From 10.0.2.2 icmp_seq=3 Destination Host Unreachable
 
--- 192.168.1.1 ping statistics ---
3 packets transmitted, 0 received, 100% packet loss, time 1999ms

The output here indicates that the remote system just can’t be reached or can’t respond. This could be the result of a routing issue or a firewall that is preventing the connection from occurring.

15.14 Step 14

Viewing the routing table would explain the error encountered in the previous step. Execute the following command to view the routing table:

route
root@ubuntu ~# route
Kernel IP routing table
Destination     Gateway         Genmask         Flags Metric Ref     Use Iface
default         _gateway        0.0.0.0         UG    100    0         0 ens3
10.0.2.0        0.0.0.0         255.255.255.0   U     0      0         0 ens3
_gateway        0.0.0.0         255.255.255.255 UH    100    0         0 ens3

The output of this command shows that the only route that exists is to hosts on the 10.0.2.0 network, in addition to the default router.

If the problem in the previous steps was due to a missing route, then one could be added temporarily for testing purposes.

15.16 Step 16

The ping6 command is used to verify connectivity with IPv6 hosts. Use ping6 with the -I option to ping all hosts on the same subnet as the local interface identified in Step 10. The -c option limits the number of pings to whatever is specified after it, otherwise it will continue indefinitely.

ping6 -I ens3 2001:9b0:104:42::102 -c3
root@ubuntu ~# ping6 -I ens3 2001:9b0:104:42::102 -c3
PING 2001:9b0:104:42::102(2001:9b0:104:42::102) from 2001:9b0:104:42::102 ens3: ens3: 56 data bytes
64 bytes from 2001:9b0:104:42::102: icmp_seq=1 ttl=64 time=0.157 ms
64 bytes from 2001:9b0:104:42::102: icmp_seq=2 ttl=64 time=0.067 ms
64 bytes from 2001:9b0:104:42::102: icmp_seq=3 ttl=64 time=0.144 ms
 
--- 2001:9b0:104:42::102 ping statistics ---
3 packets transmitted, 3 received, 0% packet loss, time 2004ms
rtt min/avg/max/mdev = 0.067/0.122/0.157/0.041 ms
Since the VM only has connectivity with itself the responses will be limited to only the ens3 interface, on a live network this command will return many additional IPv6 hosts.

15.17 Step 17

The traceroute command, commonly used for seeing how a transmission travels between a local host machine to a remote system. Use the following command to scan the route to the CentOS host at the 10.0.2.3 IP address:

traceroute 10.0.2.3
root@ubuntu ~# traceroute 10.0.2.3
traceroute to 10.0.2.3 (10.0.2.3), 30 hops max, 60 byte packets
 1 10.0.2.3 (10.0.2.3) 0.655 ms 0.307 ms 0.376 ms

Note the first 1 character in the last line of the output, this is the hop number.

15.18 Step 18

The traceroute tool can also be used for IPv6 connections. Use the traceroute6 command to view the IPv6 path to the CentOS host at the 2001:9b0:104:42::103 IPv6 address:

traceroute6 2001:9b0:104:42::103
root@ubuntu ~# traceroute 2001:9b0:104:42::103
traceroute to 2001:9b0:104:42::103 (2001:9b0:104:42::103), 30 hops max, 60 byte packets
 1 ubuntu (2001:9b0:104:42::103) 0.655 ms 0.307 ms 0.376 ms

15.19 Step 19

The tracepath command is used to trace the path to a network host, discovering MTU (maximum transmission unit) along the path. It sends ICMP and UDP messages of various sizes to find the MTU size on the path.

To trace the path of the current host, execute the following command:

tracepath 10.0.2.2
root@ubuntu ~# tracepath 10.0.2.2
1:    ubuntu                                               0.824ms reached
      Resume: pmtu 65535 hops 1 back 1

15.20 Step 20

To display the output of the tracepath command numerically (using IP addresses), use the following command:

tracepath -n 10.0.2.2
root@ubuntu ~# tracepath -n 10.0.2.2
1:    10.0.2.2                                               0.152ms reached
      Resume: pmtu 65535 hops 1 back 1

15.21 Step 21

To trace the path to the CentOS host, execute the following command:

tracepath 10.0.2.3
root@ubuntu ~# tracepath 10.0.2.3

ethtool -i ens3 | head -n 20
root@ubuntu ~# ethtool -i ens3 | head -n 20
driver: e1000
version: 7.3.21-k8-NAPI
firmware-version:
expansion-rom-version:
bus-info: 0000:00:03.0
supports-statistics: yes
supports-test: yes
supports-eeprom-access: yes
supports-register-dump: yes
supports-priv-flags: no

The output of the ethtool command above gives useful information such as the driver name and version.

15.23 Step 23

The ip neighbor command (ip neigh for short) is useful for finding other hosts on the same network as your system. Use the ip neighbor command to discover what ip addresses are reachable from the system network interface:

ip neighbor show dev ens3
root@ubuntu ~# ip neighbor show dev ens3
10.0.2.1  FAILED
10.0.2.22  FAILED
10.0.2.3 lladdr 2c:fd:a1:ac:45:32 STALE
fe80::42:acff:fe11:2 lladdr 02:42:ac:11:00:02 STALE
2001:9b0:104:42::103 lladdr ec:fd:a1:ac:45:32 STALE
fe80::2efd:a1ff:feac:4532: lladdr 2c:fd:a1:ac:45:32 STALE
fe80::42:acff:fe11:3 lladdr 02:42:ac:11:00:03 STALE

15.24 Step 24

You can change the output of ip neighbor by pinging a host. The state will become REACHABLE then go back to STALE. To view the change in the state of the the host in the ip neighbor output, use the following commands:

ip neighbor show to 10.0.2.3
ping -c5 10.0.2.3
ip neighbor show to 10.0.2.3
root@ubuntu ~# ip neighbor show to 10.0.2.3
10.0.2.3 dev ens3 lladdr 2c:fd:a1:ac:45:32 STALE
root@ubuntu ~# ping -c5 10.0.2.3
PING 10.0.2.3 (10.0.2.3) 56(84) bytes of data.
64 bytes from 10.0.2.3: icmp_seq=1 ttl=64 time=1.95 ms
64 bytes from 10.0.2.3: icmp_seq=2 ttl=64 time=1.57 ms
64 bytes from 10.0.2.3: icmp_seq=3 ttl=64 time=1.72 ms
64 bytes from 10.0.2.3: icmp_seq=4 ttl=64 time=1.53 ms
64 bytes from 10.0.2.3: icmp_seq=5 ttl=64 time=1.51 ms
 
--- 10.0.2.3 ping statistics ---
3 packets transmitted, 5 received, 0% packet loss, time 4006ms
rtt min/avg/max/mdev = 1.516/1.662/1.957/0.165 ms
root@ubuntu ~# ip neighbor show to 10.0.2.3
10.0.2.3 dev ens3 lladdr 2c:fd:a1:ac:45:32 REACHABLE

Note the state of the host is now REACHABLE, meaning that this is a valid neighbor entry until the status expires. To verify that this state is temporary and will expire, wait 30 seconds and execute the following command again:

ip neighbor show to 10.0.2.3
root@ubuntu ~# ip neighbor show to 10.0.2.3
10.0.2.3 dev ens3 lladdr 2c:fd:a1:ac:45:32 STALE

15.25 Step 25

If you suspect a problem with an interface you can use the ip link command to show the status of any interface on the system. Use the ip link command with show and the desired interface to get information about a specific network interface.

ip link show ens3
root@ubuntu ~# ip link show ens3
2: ens3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast state UP mode DEFAULT group default qlen 1000
     link/etch 2c:fd:a1:53:8b:82 brd ff:ff:ff:ff:ff:ff
