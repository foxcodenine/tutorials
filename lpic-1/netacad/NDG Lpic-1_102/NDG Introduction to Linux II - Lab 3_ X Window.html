3.1 Step 1

In this lab you will be alternating between the command line interface and the graphical user interface.

First, login to the root account. When prompted for the password, use the root password netlab123:

CentOS Linux 7 (Core)
Kernel 3.10.0-1062.el7.x86_64 on an x86_64
 
centos login: root
Password:
[root@centos ~]# _

3.2 Step 2

Next, switch to the GUI by bringing the system to runlevel 5 using the init 5 command:

init 5
[root@centos ~]# init 5

Once the GUI loads, the following login screen will appear:

3.3 Step 3

Please allow 1-2 minutes for the desktop to load.

3.4 Step 4

Use the Applications menu, System Tools menu, Terminal icon to start a terminal application.

3.5 Step 5

To perform the steps of this lab, you will need the privileges of the superuser account. Execute the following command to switch to the root account:

su –

When prompted, provide the root password netlab123:

3.6 Step 6

In runlevel 5 of CentOS, the display manager is normally running. To stop the display manager from running, but not other system services, runlevel 3 is used. Execute the following init command to switch to runlevel 3:

init 3

After you have logged out, you should be back on the command line and still logged in as the root user in the terminal.

3.7 Step 7

Log in as the root user. When prompted for a password, enter netlab123.

centos login: root
Password:
Last login Sun Jan 26 18:05:23 on pts/0
[root@centos ~]#

Next, Use the runlevel command to verify that the system was previously in runlevel 5 and is currently in runlevel 3:

runlevel
[root@centos ~]# runlevel
5 3
[root@centos ~]#

The first number 5 is the previous runlevel and the second number 3 is the current runlevel.

3.8 Step 8

Although the X server can actually run without any xorg.conf file as they automatically configure themselves when started, an xorg.conf may still be used to create custom configurations. To change the resolution the screen uses for X Windows, change the Modes entry in the /etc/X11/xorg.conf.d/screen.conf configuration file.

nano /etc/X11/xorg.conf.d/screen.conf
GNU nano 2.3.1.    File: /etc/X11/xorg.conf.d/screen.conf

Section "Screen"
  Identifier   "Screen0"
  Device       "Device0"
  Monitor      "Monitor0"
  DefaultDepth  16
  Subsection    "Display"
    Depth 16
    Modes "1024x768"
  EndSubsection
EndSection

Changes like this are the reason why you would create a /etc/X11/xorg.conf.d/screen.conf file. If you aren’t making customizations like this, then normally allowing the X server to automatically configure itself is the best option.

3.9 Step 9

In the /etc/X11/xorg.conf.d/screen.conf file, change the resolution from 1024x768 to 800x600 by modifying the Modes line to the following:

Modes "800x600"

  Identifier   "Screen0"
  Device       "Device0"
  Monitor      "Monitor0"
  DefaultDepth  16
  Subsection    "Display"
    Depth 16
    Modes "800x600"
  EndSubsection

3.10 Step 10

Instead of switching the system to runlevel 5 to switch to the graphical display, the X server and related components can be started manually. Use the startx command to test the changes to the configuration and start the Graphical User Interface (GUI):

startx
[root@centos ~]# startx

It is easier to use this technique for testing a new configuration because you are only starting the GUI, not other services that may also be started when switching to runlevel 5.

You may not see a major difference in the screen resolution. However, what is important here is that the X server starts without any error messages.
