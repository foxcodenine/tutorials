12.1 Step 1

For this lab you will be using the CentOS virtual machine.

To perform the steps of this lab you will need the privileges of the superuser account. Log in as the root user. When prompted, provide the root password netlab123:

CentOS Linux 7 (Core)
Kernel 3.10.0-957.el7.x86_64 on an x86_64
 
centos login: root
Password:
[root@centos ~]#

12.2 Step 2

Attempt to print the /etc/hosts file by executing the following command:

lp /etc/hosts
[root@localhost ~]# lp /etc/hosts                                               
lp: Error - no default destination available.

An error should occur because the CUPS service has not been started.

12.3 Step 3

Execute the following command to start the CUPS service and verify that it is running:

systemctl start cups
systemctl status cups
[root@localhost ~]# systemctl start cups                                     
[root@localhost ~]# systemctl status cups                                       
● cups.service - CUPS Printing Service                                          
   Loaded: loaded (/lib/systemd/system/cups.service; disabled; vendor preset: disabled)                                                                         
   Active: active (running) since Thu 2020-01-02 12:45:24; 0min ago             
     Docs:                                                                      
 Main PID: 57                                                                   
    Tasks: 1 (limit: 4915)                                                      
   CGroup: /system.slice/cups.service                                           
           └─57 /usr/sbin/cupsd -f                                              
                                                                                
Jan 02 12:45:24 localhost.localdomain systemd[1]: Started CUPS Printing Service

To automatically start at boot time, you could execute the following command for legacy systems:

[root@localhost ~]# chkconfig cups on

Or the following command for systemd systems:

[root@localhost ~]# systemctl enable cups

While the previous step is not required for this lab, in the real world you would want to execute this command so that the CUPS services would be turned on after each system reboot.

12.4 Step 4

Attempt to print the /etc/hosts file again by executing the following command:

lp /etc/hosts
[root@localhost ~]# lp /etc/hosts                                               
lp: Error - no default destination available.

An error should occur now because a default printer has not been created on this system

12.5 Step 5

Add a new printer to the system by executing the following lpadmin command:

lpadmin -p printer1 -E -v parallel:/dev/lp
[root@localhost ~]# lpadmin -p printer1 -E -v parallel:/dev/lp

The -p option is used to specify the printer name. The -E option is used to enable the printer to accept jobs. The -v option is used to specify the device (the file that represents the printer) that will be used. The /dev/lp device represents the parallel port. In this case, there really isn’t a printer attached to this device, but it will appear to function as a device that you can print to (even though the print jobs will never physically print).

12.6 Step 6

Make the printer1 printer the default print destination by executing the following lpadmin command:

lpadmin -d printer1
[root@localhost ~]# lpadmin -d printer1

By making this the default printer, commands like lp won’t require a printer option.

12.7 Step 7

Check the status of all printers on the system by executing the lpstat -a command:

lpstat -a
[root@localhost ~]# lpstat -a                                                   
printer1 accepting requests since Fri 03 Jan 2020 12:47:14 AM UTC

12.8 Step 8

Try to print the /etc/hosts file again by executing the following command:

lp /etc/hosts
[root@localhost ~]# lp /etc/hosts                                               
request id is printer1-1 (1 file(s))

This time the command should succeed and report the request id as the printer name concatenated with the job number: printer1-1.

12.9 Step 9

Execute the cupsreject command to prevent the printer from accepting print jobs. Execute lpstat -a to see the updated status:

cupsreject printer1
lpstat -a
[root@localhost ~]# cupsreject printer1                                         
[root@localhost ~]# lpstat -a                                                   
printer1 not accepting requests since Fri 03 Jan 2020 12:52:26 AM UTC -         
        Rejecting Jobs

12.10 Step 10

Attempt to print the /etc/passwd file by executing the following command:

lp /etc/passwd
[root@localhost ~]# lp /etc/passwd                                              
lp: Destination "printer1" is not accepting jobs.

Because the printer is rejecting jobs, the command will not succeed.

12.11 Step 11

Execute the cupsaccept command to allow the printer to start accepting print jobs, then execute the lpstat -a command again to see the updated status:

cupsaccept printer1
lpstat -a
[root@localhost ~]# cupsaccept printer1                                         
[root@localhost ~]# lpstat -a                                                   
printer1 accepting requests since Fri 03 Jan 2020 12:52:26 AM UTC

12.12 Step 12

Print the /etc/passwd file by executing the following command. Execute the lpq command to view the print jobs in the queue:

lpr /etc/passwd
lpq
[root@localhost ~]# lpr /etc/passwd                                             
[root@localhost ~]# lpq                                                         
printer1 is ready and printing                                                  
Rank    Owner   Job     File(s)                         Total Size              
active  root    1       hosts                           1024 bytes              
1st     root    2       passwd                          2048 bytes

12.13 Step 13

Execute the cupsdisable command to stop the printer from processing jobs. Execute the lpstat -a command to see that the status shows no change:

cupsdisable printer1
lpstat -a
[root@localhost ~]# cupsdisable printer1                                        
[root@localhost ~]# lpstat -a                                                   
printer1 accepting requests since Fri 03 Jan 2020 12:54:24 AM UTC

Note that the cupsreject command will not allow a print job to be sent to the print queue. Attempting to print to a print queue that is “rejecting” print jobs will result in an error.

The cupsdisable command prevents jobs from the print queue from being sent to the printer, however no error occurs when a user prints to the print queue since the print queue is still accepting print jobs. These print jobs will stay in the print queue until either the printer is enabled again or the jobs are cancelled.

12.14 Step 14

Print the /etc/cups/printers.conf file by executing the following command:

lp -d printer1 /etc/cups/printers.conf
[root@localhost ~]# lp -d printer1 /etc/cups/printers.conf                      
request id is printer1-3 (1 file(s))

Executing the lp command with the -d option explicitly specifies printer1 as the destination. Because this is the default destination, the option wasn’t necessary in this case, but was provided to demonstrate how the –d option functions.

12.15 Step 15

Execute the cupsenable command to allow the CUPS service to process the jobs again. Since a printer is not attached to the parallel device specified earlier, the jobs won't actually process and will remain in the queue. View the queue by executing the lpq command:

cupsenable printer1
lpq
[root@localhost ~]# cupsenable printer1                                         
[root@localhost ~]# lpq                                                         
printer1 is ready and printing                                                  
Rank    Owner   Job     File(s)                         Total Size              
active  root    1       hosts                           1024 bytes
1st     root    2       passwd                          2048 bytes              
2nd     root    3       printers.conf                   1024 bytes

12.16 Step 16

Execute the lprm command to remove the first print job from the queue. View the queue to verify the removal by executing the lpq command:

lprm 1
lpq
[root@localhost ~]# lprm 1                                                      
[root@localhost ~]# lpq                                                         
printer1 is ready and printing                                                  
Rank    Owner   Job     File(s)                         Total Size              
active  root    2       passwd                          2048 bytes              
1st     root    3       printers.conf                   1024 bytes

12.17 Step 17

Print the /etc/aliases file by executing the lpr -P printer1 command. Check the queue to see if the job has been added by executing lpq:

lpr -P printer1 /etc/aliases
lpq
[root@localhost ~]# lpr -P printer1 /etc/aliases                                
[root@localhost ~]# lpq                                                         
printer1 is ready and printing                                                  
Rank    Owner   Job     File(s)                         Total Size              
active  root    2       passwd                          2048 bytes              
1st     root    3       printers.conf                   1024 bytes              
2nd     root    4       aliases                         2048 bytes

Recall that the lpr command provides similar functionality to the lp command. The -P option is used to explicitly specify printer1 as the destination.

12.18 Step 18

Execute cancel to remove the print job with an id of 2. Inspect the queue again by executing the lpq command:

cancel 2
lpq
[root@localhost ~]# cancel 2                                                    
[root@localhost ~]# lpq                                                         
Rank    Owner   Job     File(s)                         Total Size              
active  root    3       printers.conf                   1024 bytes              
1st     root    4       aliases                         2048 bytes

Recall that the cancel command provides the same functionality as the lprm command.

This lab is complete.
