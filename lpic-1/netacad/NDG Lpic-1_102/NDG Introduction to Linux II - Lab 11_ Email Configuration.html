11.4 Step 4

The /etc/aliases file is not used directly by the mail system. This file is converted into a database using the newaliases command. Execute this command to rebuild the aliases database:

newaliases
[root@localhost ~]# newaliases
The command above may take 1-2 minutes to run.

If the previous command returns the error message WARNING: cannot open alias database /etc/aliases, execute the newaliases command again:

[root@localhost ~]# newaliases 
WARNING: local host name (localhost) is not qualified; see cf/README: WHO AM I? 
db_map_open(/etc/aliases.db): file changed after pre-open                       
WARNING: cannot open alias database /etc/aliases                                
Cannot create database for alias file /etc/aliases                              
[root@localhost ~]# newaliases                                                  
WARNING: local host name (localhost) is not qualified; see cf/README: WHO AM I? 
/etc/aliases: 77 aliases, longest 10 bytes, 787 bytes total

11.9 Step 9

Another way to configure forwarding of email can be done by an unprivileged (non-root) user creating a .forward file in the user’s home directory.

Start by switching users to the sysadmin user and create a ~/.forward file containing the value of root. This will forward the email of the sysadmin user to the root user. Execute the following commands to set up email forwarding:

su - sysadmin
echo root > ~/.forward
cat ~/.forward
[root@localhost ~]# su - sysadmin   
Last login: Sun Dec 15 15:37:53 UTC 2019 on console                                       
[sysadmin@localhost ~]$ echo root > ~/.forward 
[sysadmin@localhost ~]$ cat ~/.forward
root

Now that the ~/.forward file exists in the sysadmin user’s home directory, the the MTA program will send the message to the address(es) or alias(es) specified in this file; in this case the root user.

11.10 Step 10

Execute the exit command to return to the root user account:

exit
[sysadmin@localhost ~]$ exit                                                    
logout                                                                          
[root@localhost ~]#

11.11 Step 11

Send an email to sysadmin@localhost by executing the four lines of text below. Press the Enter key after typing each line, including the last line, which only contains a period . character:

mail sysadmin@localhost
Pay It Forward
Breed Success
.
Enter
[root@localhost ~]# mail sysadmin@localhost                                     
Subject: Pay It Forward                                                         
Breed Success                                                                   
.                                                                               
EOT  
You have mail in /var/spool/mail/root

You may need to wait 1-2 minutes until the You have mail in /var/spool/mail/root message appears before proceeding.

Alternatively, to verify that the /var/spool/mail/root file exists, use the ls command to view the contents of the /var/spool/mail directory:

[root@localhost ~]# ls /var/spool/mail
rpc sysadmin
[root@localhost ~]# ls /var/spool/mail
root rpc sysadmin
Note that the first command shows that the /var/spool/mail/root file did not exist yet. After waiting 1-2 minutes and executing the command again, the root file now exists in the /var/spool/mail directory.
