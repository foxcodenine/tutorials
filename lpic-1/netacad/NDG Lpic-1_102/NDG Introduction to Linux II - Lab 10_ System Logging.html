10.4 Step 4

View the rules regarding mail log files by executing the following command, which will display and number the lines containing mail from the /etc/rsyslog.conf file:

grep -n mail /etc/rsyslog.conf
[root@localhost ~]# grep -n mail /etc/rsyslog.conf                              
52:# Log anything (except mail) of level info or higher.                        
54:*.info;mail.none;authpriv.none;cron.none              /var/log/messages    
59:# Log all the mail messages in one place.                                    
60:mail.*                                                -/var/log/maillog

Lines 52 and 59 are comments that explain the rules found on lines 54 and 60. Line 54 first selects all log messages (except mail, authpriv, and cron messages) and sends these messages to the /var/log/messages file.

Line 60 is a rule that selects all mail messages and writes them to the /var/log/maillog file. The - character in front of the /var/log/maillog filename indicates to write asynchronously. Essentially, this means changes are made initially in memory and saved to the hard drive on a regular basis. This increases the performance for programs that write frequently to a log file.

10.5 Step 5

Add a new rule to the /etc/rsyslog.conf file, which will cause all mail error messages to be written to a new file: /var/log/mail.errors. Start to edit this file by executing the following command:

vi /etc/rsyslog.conf
# rsyslog v5 configuration file                                                 
                                                                                
# For more information see /usr/share/doc/rsyslog-*/rsyslog_conf.html           
# If you experience problems, see http://www.rsyslog.com/doc/troubleshoot.html  
                                                                                
#### MODULES ####                                                               
                                                                                
$ModLoad imuxsock # provides support for local system logging (e.g. via logger command)                                                                         
$ModLoad imklog   # provides kernel logging support (previously done by rklogd) 
#$ModLoad immark  # provides --MARK-- message capability                        
                                                                                
# Provides UDP syslog reception                                                 
#$ModLoad imudp                                                                 
#$UDPServerRun 514                                                              
                                                                                
# Provides TCP syslog reception                                                 
#$ModLoad imtcp                                                                 
#$InputTCPServerRun 514                                                         
                                                                                                                                    
                                                                                
"/etc/rsyslog.conf" 91L, 2875C

10.6 Step 6

Once the vi editor is started, use the G command to go to the end of the file and then the o command to enter insert mode on a new line. Then, type the text provided below:

mail.err				/var/log/mail.errors
# Save boot messages also to boot.log                                           
local7.*                                                /var/log/boot.log       
                                                                                
                                                                                
# ### begin forwarding rule ###                                                 
# The statement between the begin ... end define a SINGLE forwarding            
# rule. They belong together, do NOT split them. If you create multiple         
# forwarding rules, duplicate the whole block!                                  
# Remote Logging (we use TCP for reliable delivery)                             
#                                                                               
# An on-disk queue is created for this action. If the remote host is            
# down, messages are spooled to disk and sent when it is up again.              
#$WorkDirectory /var/lib/rsyslog # where to place spool files                   
#$ActionQueueFileName fwdRule1 # unique name prefix for spool files             
#$ActionQueueMaxDiskSpace 1g   # 1gb space limit (use as much as possible)      
#$ActionQueueSaveOnShutdown on # save messages to disk on shutdown              
#$ActionQueueType LinkedList   # run asynchronously                             
#$ActionResumeRetryCount -1    # infinite retries if host is down               
# remote host is: name/ip:port, e.g. 192.168.0.1:514, port optional             
#*.* @@remote-host:514                                                          
# ### end of the forwarding rule ###                                            
mail.err                                /var/log/mail.errors                    
-- INSERT --

Press the Esc key and type :wq followed by the Enter key to write the changes and quit:

Esc
:wq 
Enter

10.12 Step 12

On most systems logrotate is run daily as a cron job; however, it can be configured to run more often depending on factors such as log size. The file /etc/logrotate.conf determines the rotation schedule, in this example we will modify it to run whenever the size of /var/log/audit/audit.log exceeds 100kb.

Open the /etc/logrotate.conf file using the vi editor:

vi /etc/logrotate.conf
[root@localhost ~]# vi /etc/logrotate.conf

# see "man logrotate" for details
# rotate log files weekly
weekly

# keep 4 weeks worth of backups
rotate 4

#create new (empty) log files after rotating old ones
create

# use date as a suffix of the rotated file
dateext

# uncomment this if you want your log files compressed
#compress

# RPM packages drop log rotation information into this directory
include /etc/logrotate.d

# no packages own wtmp and btmp -- we’ll rotate them here
/var/log/wtmp {
   monthly
   create 0664 root utmp
       minsize 1M
“/etc/logrotate.conf” 35L, 662C

Once the vi editor is started, use the G command to go to the end of the file and then the o command to enter insert mode on a new line. Then, type the text provided below:

/var/log/audit/audit.log {
	rotate 5
	size 100k
	sharedscripts
	postrotate
	endscript
}
# system-specific logs may be also be configured here.
/var/log

Press the Esc key and type :wq followed by the Enter key to write the changes and quit:

Esc
:wq 
Enter
"/etc/logrotate.conf” 42L, 780C written
[root@localhost ~]#

The commands in the script above tell the logrotate.conf script file to keep five (rotate 5) versions of the audit.log file, start a new log file when it grows to 100kb, the sharedscripts directive means postrotate will only run once and endscript tells the script to exit.

The /etc/logrotate.d directory is where application services can store their logrotate configuration files. Most systems will already have several files in this directory created by applications that need help with rotating their log files.

10.13 Step 13

Run the logrotate.conf script in the /bin/sh shell to confirm there are no errors:

#!/bin/sh
logrotate /etc/logrotate.conf
[root@localhost ~]# #!/bin/sh
[root@localhost ~]# logrotate /etc/logrotate.conf

10.16 Step 16

To delete old systemd journal data use the journalctl command with the --vacuum-time flag to specify how much data should be preserved.

journalctl --vacuum-time=7d
[root@localhost ~]# journalctl --vacuum-time=7d
Vacuuming done, freed 0B of archived journals on disk.

This command will delete journal data older than 7 days.
